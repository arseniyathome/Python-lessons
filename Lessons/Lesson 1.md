## Git
![](https://git-scm.com/images/logo@2x.png)

Git - это система отслеживания изменений в файлах проекта, также, позволяющая легко возвращаться к предыдущим версиям.

GitHub — это сайт, где люди могут хранить свои проекты Git и совместно с ними работать.

Существуют и другие аналогичные службы (GitLab, SourceForge, сервисы Google и Amazon Web Services). Так же можно создать свой аналог, например, используя opensource-программное обеспечение.

Репозиторий — это место, где хранятся все файлы вашего проекта вместе с историей их изменений.

# Командная сnрока
Запустим интерпретатор команд Windows. 

`Пуск -> Выполнить (или Win+R)`
Введем:
`cmd`

В приглашение командной строки мы видим папку, в которой мы сейчас находимся. В Windows папки расположена на дисках. Чтобы сменить диск введем `c:` или `d:`.

Чтобы сменить папку, используется команда `cd`. Чтобы создать папку - `mkdir`. Выполним следующие команды:

`d:` - перейдем на нужный нам диск (можно указать другую букву, в зависимости от конфигурации Вашей системы)

`cd \` - переместимся в корень диска

`mkdir Projects` - создадим папку на диске

`cd Projects` - перейдем в созданную папку 

`dir` - посмотрим содержимое папки. Она пока пуста.

Снова выведем содержимое командой `dir`. Появилась папка Python-lessons, переместимся в создавшуюся папку `cd Python-lessons`. Посмотрим что в ней. `dir` - ура, мы получили нужные данные на наш компьютер!

# Еще немного о командной строке
`cd ..` - переход в вышестоящую папку. Например, применив эту команду в папке d:\Projects\Python-lessons, мы окажемся в папке d:\Projects

Для повтора уже введенных команд можно использовать клавишу "Вверх". Если нажимать ее несколько раз, то будут передираться все ранее введенные команды.

Для ускорения ввода можно использовать клавишу Tab. Перейдем в папку Projects, наберем `cd` и попробуем использовать Tab. 

Так же можно ввести первые буквы имени файла или папки. Перейдем в корень диска: `cd \`, наберем `cd p` и попробуем понажимать Tab несколько раз.

Для ввода имен, включающих в себя пробелы, их следует заключать в кавычки, например `cd "c:\Program files". Важно отметить, что в Windows регистр не имеет значения.

# Far Manager
Для более удобной и визуализированной работы с файлами можно использовать интерфейс Windows. Однако существуют и альтерналивные способы. Это использование файловых менеджеров. Рассмотри работу с Far Manager.

Установку можно произвести, скачав дистрибутив с официального сайта.

[https://www.farmanager.com/download.php?l=ru](https://www.farmanager.com/download.php?l=ru)

Far позволяет работать с файлами, папками, архивами, имеет встроенный редактор и средства просмотра, умеет работать с сетевыми ресурсами и многое другое. Функционал можно расширить с помощью плагинов и скриптов. 

Интерфейс состоит из двух панелей. Переключаться между ними можно при помощи клавиши Tab. Вот другие полезные команды:

`Alt+F1, Alt+F2` - выбор диска в левой или правой панели

`F4` - редактировать файл

`Shift-F4` - создать новый файл

`F3` - просмотреть файл

`Alt+Q` - быстрый просмотр файла

`F7` - создать папку

`Ins` - выделить файл или папку или снять выделение

`+, -` - выделить или снять выделение по маске

`F5` - скопировать выбранные файлы

`F6` - переместить выбранные файлы

`Shift+F6` - переименовать 

# Файл .gitignore
При работе с проектом часто возникают файлы, которыми не нужно делиться с коллегами и не следует отправлять в удаленный репозиторий на GutHub. Например среда разработки Pycharm компилирует написанные программы перед запуском и хранит их в специальных папках. Эти файлы могут работать неправильно у других участников проекта, напрмер, в разых операционных системах или из-за различий в локальных настройках. Поэтому некоторые объекты следует исключать. 

Для этого в системе Git предусмотрен специальный файл .gitignore, который расположен в корне проекта. Вот так примерно он выглядит в нашем учебнике:

> .idea/   
> **/__pycache__/  
> *.pyc  

В каждой строке перечислены пути и папки. В первой - служебные данные Pycharm. Во второй  - папки содержащие скомпилированный код, в третьей - все файлы в любых папках с расширением pyc.

Теперь система Git не будет отправлять указанные объекты.

# Первоначальная настройка Git
Это очень простая операция. Нужно задать два глобальных параметра, которые будкт использоваться во всех проектах на компьютере:

> git config --global user.name "John Doe"  
> git config --global user.email johndoe@example.com  

Естественно нужно указать свое имя и e-mail, указанный при регистрации на GitHub.

# Файлы разметки md и html
Для оформления web-страниц в интернете используется специальный язык HTML (Hypertext markup language - язык разметки гипертекста). Если посмотреть исходный код любой страницы в Интернете, нажав в Chrome Ctrl+U, мы увидим что-то такое:
```
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Пример страницы</title>
    <style>
        body {
            font-family: Arial, sans-serif; /* Устанавливаем шрифт для всей страницы */
            background-color: #f4f4f4; /* Цвет фона страницы */
            margin: 20px; /* Отступ от краев браузера */
        }
        h1 {
            color: #333; /* Цвет текста заголовка */
        }
        p {
            color: #666; /* Цвет текста параграфа */
            line-height: 1.6; /* Высота строки текста */
        }
    </style>
</head>
<body>
    <h1>Заголовок страницы</h1>
    <p>Это пример страницы с использованием HTML и CSS. Здесь вы можете видеть, как можно  стилизовать текст и другие элементы на веб-странице.</p>
</body>
</html>
```
А выглядеть в броузере такой код будет всего лишь так:

![](images/html-example.png)

Конечно язык HTML позволяет оформлять очень крутые страницы, но для написания многих типов документов он неудобен. Поэтому был придуман формат MD (Markdown). По сути это просто текстовый файл, в который добавляются некоторые специальные символы, позволяющие оформить текст. 

Например:

`#` или `##` в начале строки превращают их в заголовки.  

Символ `>` позволяет выделить в тексте исходный код.  

Конструкция `[Ссылка на сайт](https://my-site-adress.com)` отображает гиперссылку. И так далее. 

В качестве примера разметки MD можно использовать этот документ. Файлы такого типа  рендерят многие системы: Pycharm и VS Code сразу умеют показывать превью, GitHub автоматически отображает их в красивом виде.
